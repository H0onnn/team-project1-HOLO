1.
CONN : 접속 명령
SYSTEM/1111 : SYSTEM계정/비밀번호1111
SYSTEM계정 : ORACLE 설치하면 자동으로 존재하는 관리자 계정
참고 : SYSTEM, SYS 계정이 관리자 계정이다.
비밀번호1111 : ORACLE 설치할 때 입력하는 창이 나타난다.

처음에 관리자 계정으로 접속해야만 새로운 사용자를 만들 수 있다.
새로운 사용자를 만드는 권한은 관리자 밖에 없다.

2. 
새 사용자 만들기
CREATE USER 사용자이름 IDENTIFIED BY 비밀번호;

3.
사용자에게 DBA(관리자) 권한을 주기
GRANT 권한종류 TO 사용자
- GRANT CONNECT TO JAVAWEB0525;    접속 권한을 준다.
- GRANT CONNECT, RESOURCE TO JAVAWEB0525;     접속 권한 + 자원 사용 권한을 준다.

학습의 편의를 위해서 DBA 권한을 주었다. (모든 걸 다 할 수 있는 최상위 권한)

4.
새 사용자로 재접속하기

CONN JAVAWEB0525/1111

-------
연습

관리자 계정으로 접속하기
CONN SYSTEM/1111

사용자 권한 해제하기
REVOKE 권한 FROM 사용자;

사용자 제거하기
- 사용자가 가진 게 없을 때
DROP USER 사용자;
- 사용자가 가진 게 있을 때 (테이블 등)
DROP USER 사용자 CASCADE;

관계형 데이터베이스
테이블 (표)

테이블 만들기 = 열 만들기 (칼럼, 필드, 속성, 프로퍼티)

열 만들기에서 고려할 사항
1. 어떤 데이터를 저장하는가? (숫자? 문자? 날짜?)
2. 크기는 얼마인가? (최대 20자? 숫자 5자리?)
3. 널 유무? (공백이 가능한가?)
4. 중복 가능 유무?
등등

오라클 데이터 타입

종류	오라클		자바
문자	VARCHAR2	String
숫자	NUMBER		int, double
날짜	DATE		java.sql.Date 클래스	

널 유무
1. 널 가능(필수 아님) : NULL (또는 생략)
2. 널 불가능(필수임) : NOT NULL

중복 가능 여부
중복이 불가능한 경우(아이디, 이메일 등) : UNIQUE

- 회원테이블 (MEMBER)
1. 아이디 : ID, 가변길이문자 20자, 필수, 중복 불가
2. 비밀번호 : PW, 가변길이문자 20자, 필수
3. 이름 : NAME, 가변길이문자 20자, 필수
4. 나이 : AGE, 숫자
5. 주소 : ADDRESS, 가변길이문자 100자
6. 핸드폰 : HP, 가변길이문자 13자, 중복 불가
7. 가입일 : REGDATE, 날짜

- 테이블 구조 확인
DESC 테이블
이름, 널 유무, 타입 확인 가능
--- 참고 : 내림차순 정렬시 사용하는 DESC 과 같은 모양의 키워드이지만, 둘은 전혀 다르다.
- DESC 테이블 (DESCRIBE)
- ORDER BY 필드 DESC (DESCENDING) 내림차순 정렬

- 테이블 삭제
DROP TABLE 테이블;


- 기본키(식별자, PRIMARY KEY)
- 회원들의 구분을 무엇으로 할 것인가? 아이디


- 기본키(식별자, PRIMARY KEY)의 조건
1. 널 값이 없어야 한다.
2. 중복이 없어야 한다.
3. 예 : 주민등록번호, 운전면허번호, 각종 일련번호, 학번, 군번, 제품코드 등
